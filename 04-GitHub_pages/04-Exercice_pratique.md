# Exercice pratique

1. **Init** : Un des deux crée un repository sur GitHub
2. **Collab** : Il ajoute l’autre dans les collaborateurs
3. **Clone** : Chacun clone le repository en local
4. **Branch** : Depuis VSCode, chacun crée une branche `prenom`
5. **Commit** : **_Création_** de fichiers, ajout et commit
6. **Push** : Publier sur GitHub la branche créée
-----
1. **Pull** : Récupération la branche de son·a camarade en local
2. **Switch** : Basculer sur la branche de son·a camarade
3. **Commit** : Chacun effectue un commit dans la branche
4. **Push** : Chacun push la branche modifiée sur le repo distant
5. **_Pull Request_** : Création d’une PR de la branche vers main
6. **Review** : Chacun passe en revue la PR de l’autre et merge
-----
1. **Fork** : L’un crée un _fork_ du _repository_ d’un autre groupe
2. **Branch** : Chacun du binôme crée une branche `prenom` depuis main sur son _fork_
3. **Commit** : chacun effectue un commit dans sa branche sur LES LIGNES DES MÊMES FICHIERS
4. **Push** : chacun push sa branche sur son _fork_
5. **_Pull Request_** : Création d’une PR de chaque branche des _forks_ vers main du _repository_ d’origine
6. **Review** : Chaque binôme passe en revue la PR de l'autre groupe et en merge une seule

> ⚠️ L'autre Pull Request ne peut plus être mergée car elle présente un conflit !
-----
1. **Sync** : Récupération de la branche main du _repository_ d’origine sur son fork sur Github
2. **Pull** : Récupération de main du _fork_ en local
3. **Merge** : Fusion de main dans sa branche `prenom` (résolution des conflits si besoin)
4. **Push** : Pousser la branche `prenom` mise à jour sur le _fork_
5. **Fusion** : Sur la page de _Pull Request_, finaliser la fusion
